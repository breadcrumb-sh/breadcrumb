---
title: Project Memory
---

The monitor gets smarter the longer you use it. As it analyzes your traces, it builds up memory about your project - what's normal, what's noise, what you actually care about.

A retry loop in your search agent might be expected. A cost spike after a deploy might be intentional. The monitor learns these patterns so it stops flagging them and focuses on real problems.

You can view and edit memory in your project settings. Add context the monitor hasn't learned yet, correct things it got wrong, or tell it what to pay attention to.
