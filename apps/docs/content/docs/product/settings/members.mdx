Members are managed at the organization level. Invite team members and assign roles to control access.

## First setup (self-hosted)

The first person to sign up becomes the **global admin**. This account can access all projects and organizations.

After that, every new signup requires an invitation. The global admin can invite others from the project settings.

## Roles

| Role | Permissions |
|------|-------------|
| **Owner** | Full access, can delete project |
| **Admin** | Manage members, keys, settings |
| **Member** | View and interact with traces, explore |
| **Viewer** | Read-only access to traces and dashboard |

The global admin bypasses all role checks and can access any project.
